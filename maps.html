Array Method 01

.map()

Transform everything. Touch nothing.

.map() takes an array, runs a function on EVERY item, and gives you back a brand new array of the same length — but with each item transformed. The original array is never modified.

Think of it like a factory conveyor belt. Every item goes in, gets processed, and comes out the other end changed. Nothing is skipped, nothing is removed. One item in = one item out.

Syntax

.map() — Anatomy
const result = array.map((item, index, originalArray) => {
  // item          → the current element being processed
  // index         → its position (0, 1, 2...) — optional
  // originalArray → the original array itself  — optional
  return /* transformed version of item */;
});

Simple Examples

map() Examples
const numbers = [1, 2, 3, 4, 5];

// Double every number
const doubled = numbers.map(n => n * 2);
// → [2, 4, 6, 8, 10]

// Convert to strings
const asStrings = numbers.map(n => `Item #${n}`);
// → ["Item #1", "Item #2", "Item #3", "Item #4", "Item #5"]

// With objects — extract just one field
const users = [
  { id: 1, name: 'Alice', age: 28 },
  { id: 2, name: 'Bob',   age: 34 },
  { id: 3, name: 'Carol', age: 22 },
];

const names = users.map(user => user.name);
// → ["Alice", "Bob", "Carol"]

// Transform objects — add a new field to each
const withSenior = users.map(user => ({
  ...user,
  isSenior: user.age >= 30
}));
// → [{ id:1, name:'Alice', age:28, isSenior:false }, ...]

The Most Important Use of .map() in React

In React, .map() is how you render lists. Whenever you have an array of data and want to render a component for each item, you use .map():

.map() in React JSX — The Core Pattern
const servers = [
  { id: 1, name: 'Database',   status: 'Online'  },
  { id: 2, name: 'Web Server', status: 'Offline' },
  { id: 3, name: 'Cache',       status: 'Online'  },
];

function ServerList() {
  return (
    <ul>
      {servers.map((server) => (
        <StatusItem
          key={server.id}       {/* key is REQUIRED — helps React track items */}
          name={server.name}
          status={server.status}
        />
      ))}
    </ul>
  );
}
// Each server object → one <StatusItem /> component
// 3 servers → 3 components rendered. Automatically.
What .map() Returns

Always a new array of the same length. Every item in the original array produces exactly one item in the result. If your array has 100 items, your result has 100 items. The original array is untouched.

⚠️ The key prop

In React, when you use .map() to render a list, every element MUST have a unique key prop. React uses this to track which items changed, were added, or removed. Use a unique ID from your data — never use the index as key if the list can change order.

Array Method 02

.filter()

Keep what passes. Discard what doesn't.

.filter() tests every item against a condition. If the condition returns true, the item is kept. If false, it's removed. The result is a new array with only the items that passed the test.

Think of it like a bouncer at the door. Every item walks up. The bouncer checks the condition. Pass? You're in. Fail? You're out. The bouncer never changes you — just decides in or out.

Syntax

.filter() — Anatomy
const result = array.filter((item, index, originalArray) => {
  // Must return true  → item is KEPT in the result
  // Must return false → item is REMOVED from the result
  return /* boolean condition */;
});

Simple Examples

filter() Examples
const numbers = [1, 5, 12, 3, 18, 7, 25];

// Keep only numbers greater than 10
const big = numbers.filter(n => n > 10);
// → [12, 18, 25]

// Keep only even numbers
const evens = numbers.filter(n => n % 2 === 0);
// → [12, 18]

// With objects — keep online servers
const servers = [
  { id: 1, name: 'Database',   status: 'Online'      },
  { id: 2, name: 'Auth Server', status: 'Offline'     },
  { id: 3, name: 'Cache',       status: 'Online'      },
  { id: 4, name: 'Proxy',       status: 'Maintenance' },
];

const onlineOnly = servers.filter(s => s.status === 'Online');
// → [{ id:1, name:'Database', ... }, { id:3, name:'Cache', ... }]

// Remove an item by ID (common React pattern!)
const withoutId2 = servers.filter(s => s.id !== 2);
// → All servers EXCEPT the one with id:2

// Filter strings by search term
const fruits = ['apple', 'apricot', 'banana', 'avocado'];
const aFruits = fruits.filter(f => f.startsWith('a'));
// → ["apple", "apricot", "avocado"]

.filter() in React — Search, Delete, Tabs

.filter() in React — Real Patterns
import { useState } from 'react';

function ServerDashboard() {
  const [servers, setServers] = useState([
    { id: 1, name: 'Database', status: 'Online'  },
    { id: 2, name: 'Cache',    status: 'Offline' },
    { id: 3, name: 'Proxy',    status: 'Online'  },
  ]);
  const [tab, setTab] = useState('All');
  const [search, setSearch] = useState('');

  // Pattern 1: Filter by tab
  const byTab = tab === 'All'
    ? servers
    : servers.filter(s => s.status === tab);

  // Pattern 2: Filter by search input
  const visible = byTab.filter(s =>
    s.name.toLowerCase().includes(search.toLowerCase())
  );

  // Pattern 3: Delete item from state (immutably!)
  const deleteServer = (id) => {
    setServers(prev => prev.filter(s => s.id !== id));
    // Creates new array WITHOUT the deleted item
  };

  return (
    <div>
      <input value={search} onChange={e => setSearch(e.target.value)} />
      {visible.map(s => <ServerCard key={s.id} {...s} onDelete={deleteServer} />)}
    </div>
  );
}
What .filter() Returns

Always a new array — but potentially shorter. If no items pass the test, you get an empty array []. If all items pass, you get a copy of the whole array. The original is never changed. Items are never modified — only included or excluded.

Array Method 03

.reduce()

Many items. One answer.

.reduce() is the most powerful and most misunderstood of the three. It processes every item in an array and accumulates them into a single output value. That value can be a number, string, object, array — literally anything.

Think of reduce like a snowball rolling down a hill. It starts small (the initial value), picks up snow (each array item) as it rolls, and by the time it reaches the bottom it's become one big ball (the final result).

Syntax — The Key to Understanding reduce()

.reduce() — Anatomy
const result = array.reduce((accumulator, item, index, originalArray) => {
  // accumulator → the "running total" / result so far
  // item        → the current element being processed
  // index       → current position (optional)
  // The value you RETURN becomes the next accumulator
  return /* updated accumulator */;
}, initialValue);
// initialValue → what accumulator starts as (0, '', [], {}, etc.)

Step-by-Step Trace — How reduce() actually runs

Tracing reduce() execution step by step
const numbers = [10, 20, 30, 40];

const total = numbers.reduce((acc, item) => acc + item, 0);

// Step 0: acc = 0  (initialValue),  item = 10  →  returns 0  + 10 = 10
// Step 1: acc = 10,                 item = 20  →  returns 10 + 20 = 30
// Step 2: acc = 30,                 item = 30  →  returns 30 + 30 = 60
// Step 3: acc = 60,                 item = 40  →  returns 60 + 40 = 100
// No more items. Final acc = 100. That's what reduce returns.

console.log(total); // 100

reduce() can do anything

Sum & Product
const nums = [5, 10, 15, 20];

// Sum all numbers
const sum = nums.reduce(
  (acc, n) => acc + n, 0
);
// → 50

// Find maximum
const max = nums.reduce(
  (acc, n) => n > acc ? n : acc, 0
);
// → 20

// Count occurrences
const words = ['a', 'b', 'a', 'c', 'a'];
const count = words.reduce((acc, w) => {
  acc[w] = (acc[w] || 0) + 1;
  return acc;
}, {});
// → { a:3, b:1, c:1 }
Build Objects & Flatten
// Build a lookup object from array
const users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob'   },
];
const byId = users.reduce((acc, user) => {
  acc[user.id] = user;
  return acc;
}, {});
// → { 1: {id:1,name:'Alice'}, 2: ... }
// Now: byId[1].name → 'Alice' instantly!

// Flatten nested arrays
const nested = [[1,2], [3,4], [5]];
const flat = nested.reduce(
  (acc, arr) => [...acc, ...arr], []
);
// → [1, 2, 3, 4, 5]

.reduce() in React

.reduce() in React — Practical Uses
const servers = [
  { id: 1, name: 'Database', status: 'Online',  requests: 120 },
  { id: 2, name: 'Cache',    status: 'Online',  requests: 80  },
  { id: 3, name: 'Proxy',    status: 'Offline', requests: 0   },
];

// Total requests across ALL servers
const totalRequests = servers.reduce(
  (acc, s) => acc + s.requests, 0
);
// → 200

// Group servers by status
const grouped = servers.reduce((acc, s) => {
  if (!acc[s.status]) acc[s.status] = [];
  acc[s.status].push(s);
  return acc;
}, {});
// → { Online: [Database, Cache], Offline: [Proxy] }

// Use in JSX for summary stats
function Dashboard({ servers }) {
  const stats = servers.reduce((acc, s) => ({
    total:    acc.total + 1,
    online:   acc.online   + (s.status === 'Online'  ? 1 : 0),
    requests: acc.requests + s.requests,
  }), { total: 0, online: 0, requests: 0 });

  return <p>{stats.online} / {stats.total} online · {stats.requests} req/s</p>;
}
What .reduce() Returns

Whatever the accumulator is at the end. It's completely up to you — a number, string, boolean, object, array, anything. That's what makes reduce so powerful and flexible. You define the shape of the output by setting the initialValue and what you return each step.

When to Use What

The Decision
Guide

"Ask yourself: do I want to transform it, trim it, or total it?"
.map()

Use when you want to:

  • Render a list of components
  • Transform every item
  • Extract a field from objects
  • Add a property to each object
  • Convert types (number → string)
  • Apply formatting to all items
.filter()

Use when you want to:

  • Remove items from a list
  • Implement search / filtering UI
  • Show only "Online" servers
  • Delete an item from state
  • Show items matching a category
  • Remove duplicates or nulls
.reduce()

Use when you want to:

  • Sum / average numbers
  • Count items by category
  • Build an object from an array
  • Group items (by status, type…)
  • Flatten nested arrays
  • Compute multiple stats at once
Key Differences

The "Does it change the array length?" test

Question Answer Use
Do I want the SAME number of items, but different? Yes, just transformed .map()
Do I want FEWER items based on a condition? Yes, some removed .filter()
Do I want ONE result from ALL the items? Yes, collapsed to one value .reduce()

Return values side by side

Same array. Three methods. Three different outputs.
const prices = [10, 25, 5, 40, 15];

// .map()   → New array, same length, every item changed
prices.map(p => p * 1.1);
// → [11, 27.5, 5.5, 44, 16.5]  ← 5 items, all transformed

// .filter() → New array, fewer items, nothing changed about items
prices.filter(p => p > 10);
// → [25, 40, 15]               ← 3 items, identical values

// .reduce() → Single value (not an array)
prices.reduce((acc, p) => acc + p, 0);
// → 95                         ← one number, not an array at all

Chaining — Real World Example
const servers = [
  { id: 1, name: 'Database', status: 'Online',  requests: 120, memory: '70%' },
  { id: 2, name: 'Cache',    status: 'Offline', requests: 0,   memory: '10%' },
  { id: 3, name: 'Proxy',    status: 'Online',  requests: 85,  memory: '45%' },
  { id: 4, name: 'Auth',     status: 'Online',  requests: 200, memory: '80%' },
];

// Goal: Get total requests from ONLINE servers only
const totalOnlineRequests = servers
  .filter(s => s.status === 'Online')   // Step 1: keep Online only [DB, Proxy, Auth]
  .map(s => s.requests)                   // Step 2: extract requests [120, 85, 200]
  .reduce((acc, r) => acc + r, 0);       // Step 3: sum them → 405

// In React JSX — filter first, THEN map to render
function Dashboard({ servers, showOnlyOnline }) {
  return (
    <div>
      {servers
        .filter(s => showOnlyOnline ? s.status === 'Online' : true)
        .map(s => <StatusItem key={s.id} {...s} />)
      }
    </div>
  );
}
Important Order Rule

Always .filter() before .map() when chaining. Filtering first reduces the number of items, so .map() has less work to do. Doing it the other way works but is wasteful — you'd transform items that then get thrown away.

In Your Project

Putting It All
Together

Here's a complete, realistic example using all three methods in one React component — exactly the kind of code you'd write in your server dashboard:

ServerDashboard.jsx — All Three Methods Working Together
import { useState } from 'react';

function ServerDashboard() {
  const [servers] = useState([
    { id: 1, name: 'Database', status: 'Online',  requests: 120 },
    { id: 2, name: 'Cache',    status: 'Online',  requests: 80  },
    { id: 3, name: 'Proxy',    status: 'Offline', requests: 0   },
    { id: 4, name: 'Auth',     status: 'Online',  requests: 200 },
  ]);
  const [filter, setFilter] = useState('All');

  // ── .filter() → Only show servers matching selected tab ──
  const visibleServers = servers.filter(s =>
    filter === 'All' ? true : s.status === filter
  );

  // ── .reduce() → Summary stats from all servers ──────────
  const stats = servers.reduce((acc, s) => ({
    totalRequests: acc.totalRequests + s.requests,
    onlineCount:   acc.onlineCount + (s.status === 'Online' ? 1 : 0),
  }), { totalRequests: 0, onlineCount: 0 });

  return (
    <div>
      <{/* Stats bar — powered by .reduce() */}>
      <p>{stats.onlineCount} online · {stats.totalRequests} total req/s</p>

      <{/* Filter tabs — drives .filter() above */}>
      {['All', 'Online', 'Offline'].map(tab => (    {/* .map() renders the tabs! */}
        <button key={tab} onClick={() => setFilter(tab)}>{tab}</button>
      ))}

      <{/* Server list — powered by .filter() + .map() */}>
      {visibleServers.map(server => (           {/* .map() renders each card */}
        <StatusItem key={server.id} {...server} />
      ))}
    </div>
  );
}
One-Line Summaries
.map() "I want the same number of items, but each one transformed into something else." → Array (same length)
.filter() "I want fewer items — only the ones that match my condition." → Array (shorter)
.reduce() "I want to collapse the whole array into one single value." → Anything

None of them mutate the original array. Ever.
